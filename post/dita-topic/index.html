DITA Topic

2023-06-04

学习目标：
 识别通用Topic中的常用元素。
 引用DITA中定义的标准主题类型。
 标识不同类型Topic中通常可以找到的元素。
 定义专门化和约束。

标准DITA Topic类型

 Concept——包含概念信息，回答“为什么”的问题。
 Task——包含分布过程信息，回答“如何做”的问题。
 Reference——包含参考信息，回答“是什么”的问题。
 Glossary Entry——定义单个术语，回答“这是什么意思”的问题。

通用型Topic
所有DITA Topic都具有以下这种基础性结构：
唯一必要的元素。<br> <Shortdesc> （简短描述）元素，通常用2-3句总结来描述。<br> <body> 包含topic 的大部分信息，在特定topic类型中，body元素有自己特有的标签，例如概念性topic的<conbody>和任务型topic的<taskbody>。</p> <p><strong>Concept 类型的Topic</strong><br>  一个Concept Topic 回答“为什么”的问题。她为读者正要了解的主题提供背景信息。<br>  Concept Topic 同行包含文本段落和列表，但也可包含注释，表格和图形元素，这些都是理解特定主题背后的想法所需要的。<br> Concept Topic中使用的常见元素包括：<br> <conbody> Concept Topic的正文<br> <p> 一个段落<br> <ul> 一个无序列表或者项目符号列表<br> <ol> 一个有序列表或者编号列表<br> <li> 内嵌于<ul>或<ol>里的一个列表项<br> <fig> 图片，包含一个可选标题<br> <image> Figure里面的图片或文本行间的图片<br> <section> Topic 的子章节，带有可选标题</p> <p><strong>Task类型的Topic</strong><br> Task Topic回答怎么做的问题，它包括完成过程的步骤说明。DITA还允许包括步骤结果、图形、注释和以及子步骤等元素。</p> <p>技术内容通常是由众多Task组成，DITA提供两种类型的Task:<br> 严格的类型 (Strict Task)：要求所有元素以特定的顺序出现，只允许一个<example>元素，并且对于步骤只能选用两个非常正式的元素（<steps>或<steps-unordered>）.<br> 一般的类型 (General Task)：也称为松散任务，在元素排列的顺序上更为灵活，允许多个<example>元素，并且允许使用非正式步骤元素（<steps-informal>），它可以包含更多不同的内容。<br> Strict Task 适用于需要分布说明的项目；General Task对过程概述很有用，对于大多数技术文档而言，DITA的最佳时间建议大家尽量使用Strict Task，<br> Strict Task 中使用的常见元素包括：<br> <taskbody> Task的正文<br> <steps>任务过程，即一个步骤<br> <step>任务中的每个单独步骤<br>  <cmd> 用户采取的动作；这是<step>中的必要元素<br>  <info> 关于步骤的附加信息<br>  <stepresult> 执行操作后会发生什么<br>  <stepxmp> 举例说明是如何做这个步骤<br> <example> 举例说明如何做整个Task<br> Reference 类型的Topic 中使用的常见元素包括：<br> Reference类型的Topic 回答是什么的问题。Reference 类型的Topic 通常包含描述性事实，例如命令或API函数调用的语法，以表格形式劣处设备的操作特性个容县范围，或识别软件屏幕上的信息项。</p> <p>Reference 不包括步骤或背景信息。Reference 与Glossary Entry类似，金提供事实。</p> <p><strong>Reference Topic中使用的常见元素包括：</strong><br> <refbody> Reference 的正文<br> <section> Reference 的子章节，带有可选标题<br> <table> 表格<br> <properties> 属性列表<br> <refsyn> 语法图</p> <p><strong>Glossary Entry 类型的Topic</strong><br> Glossary Entry类型的Topic回答这个词组或短语是什么意思的问题。Glossary Entry 通常包含一个术语，一集一个或多个定义。<br> Glossary Entry中使用的常见元素包含：<br> <glossentery> Glossary Entry topic的类型<br> <glossterm> 单词或词组<br> <glossdef> 词汇表术语的定义</p>
